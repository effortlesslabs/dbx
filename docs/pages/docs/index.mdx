# Introduction

DBX is a minimal, blazing-fast API proxy designed for edge and embedded systems. It provides a unified API layer to expose Redis, Qdrant, and MDBX databases through a clean, RESTful interface and WebSocket connections.

## What is DBX?

DBX acts as a lightweight proxy that sits between your applications and your databases, providing:

- **Unified API**: Single interface for multiple database types
- **Edge-Optimized**: Designed for low-latency, high-throughput edge deployments
- **Embedded-Friendly**: Minimal resource footprint perfect for embedded systems
- **Protocol Support**: Both REST API and WebSocket connections
- **Type Safety**: Full TypeScript SDK with comprehensive type definitions

## Use Cases

### Edge Computing

Deploy DBX on edge nodes to provide local database access with cloud synchronization capabilities.

### IoT Applications

Use DBX in embedded systems to provide a standardized API for various database backends.

### Microservices

Integrate DBX as a database proxy layer in microservice architectures for consistent data access patterns.

### Real-time Applications

Leverage WebSocket connections for real-time data synchronization and event streaming.

## Quick Start

Get started with DBX in minutes:

```bash
# Install DBX
cargo install dbx

# Start the server
dbx --config config.toml

# Use the TypeScript SDK
npm install @effortlesslabs/dbx
```
